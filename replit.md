# LocalModel Studio

A self-contained AI chat app that runs open-source language models entirely in the browser via WebGPU (WebLLM), packaged as an installable PWA, with a marketing/download landing page.

## Artifacts in this project

- `artifacts/landing/` — marketing & download site, served at `/`. One-page dark scrolling site with a scroll progress bar, animations, and a PWA install CTA.
- `artifacts/localmodel-studio/` — the actual chat app, served at `/app/`. PWA-installable.
- `artifacts/api-server/` — scaffold; not used (the app is fully client-side).
- `artifacts/mockup-sandbox/` — canvas/mockup sandbox; not used in production.

## Run & Operate

- `pnpm --filter @workspace/landing run dev` — marketing site
- `pnpm --filter @workspace/localmodel-studio run dev` — chat app
- `pnpm run typecheck` — full typecheck across all packages

## PWA / Download story

The chat app is installable as a Progressive Web App — this is the "download" experience. Users click "Install App" on the landing page and the browser registers it as a native-like app on their device (dock/start menu icon, standalone window, offline-capable shell).

- `artifacts/landing/public/manifest.webmanifest` — PWA manifest. `start_url` is `/app/` so the installed app launches the chat UI.
- `artifacts/landing/public/sw.js` — minimal service worker (required for install prompt). Network-first; caches the shell.
- `artifacts/landing/src/hooks/usePWAInstall.ts` — handles the `beforeinstallprompt` event, exposes install state for the CTA button. Detects iOS (manual A2HS needed) and unsupported browsers.
- Icons (`pwa-192.png`, `pwa-512.png`, `pwa-512-maskable.png`, `apple-touch-icon.png`, `favicon.svg`) are generated by `scripts/src/generate-pwa-icons.ts` (sharp) and live in both `artifacts/landing/public/` and `artifacts/localmodel-studio/public/`.

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- Frontend: React + Vite + Tailwind CSS v4 + shadcn/ui + framer-motion + wouter
- Inference: `@mlc-ai/web-llm` in the app (in-browser WebGPU, fully offline after first model download)
- Storage: localStorage for conversations/profiles; browser cache (IndexedDB) for model weights
- No backend API, no external runtimes

## Where things live (chat app)

- `artifacts/localmodel-studio/src/pages/` — Dashboard, Chat, Models, Settings
- `artifacts/localmodel-studio/src/services/` — webllmService, storageService, systemInfo
- `artifacts/localmodel-studio/src/components/` — shared UI components
- `artifacts/localmodel-studio/src/index.css` — theme / CSS variables

## Architecture decisions

- **Frontend-only, self-contained app**: Pure SPA. No backend, no external runtimes.
- **In-browser inference**: All inference runs locally via WebLLM on WebGPU. The model file is fetched from a CDN the first time, then cached and runs fully offline.
- **PWA-installable**: Service worker + manifest at the root (landing) artifact. Install prompt triggered from the marketing site via `beforeinstallprompt`.
- **Two artifacts, same origin**: Landing site at `/`, app at `/app/`. Both served as static SPAs through the shared proxy.
- **Dark mode by default**: `dark` class added to `document.documentElement`. Landing site is dark-only.
- **Profile migration**: One-time migration in `storageService.getModelProfiles()` maps legacy Ollama IDs to WebLLM equivalents or drops unmappable profiles, then persists the cleaned list.

## Product

LocalModel Studio gives developers and AI enthusiasts a clean, private UI to:
- Chat with open-source LLMs running entirely in the browser
- Manage model profiles (sampling params, context length)
- See system info, WebGPU status, and model recommendations
- Store all conversations locally — nothing leaves the device

## User preferences

- UI should be modern and user friendly, but simple and not too much going on
- No third-party tools or external installs — the app must be a single, self-contained package
- Landing page is dark, modern, scroll-driven, animated, with a top scroll-progress bar and non-templated features

## Gotchas

- WebGPU is required to run models — Chrome 113+ or Edge 113+ on a device with a supported GPU
- First model load needs internet to download weights (~0.7–5 GB depending on model). After that the model runs fully offline.
- PWA install prompt only fires on browsers that support it (Chrome/Edge desktop & Android). iOS Safari uses manual "Add to Home Screen"; Firefox desktop doesn't support install prompts — the landing page handles both fallbacks.
- The chat app defaults to dark mode; light mode can be toggled from settings.
